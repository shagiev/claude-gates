# Review guidelines для Codex (claude-gates — репо самого плагина gates)

Ты — независимый ревьюер. Оценивай изменения по инвариантам проекта. Severity
`critical`/`high` = блокирует деплой — ставь только при реальном риске.

**Если ревьюишь ДИЗАЙН/ПЛАН** — суди полноту сценариев (секция BSAC): перечислены ли
топ-уровневые сценарии, закрыт ли каждый fail-open краевой случай, что упущено.

## Blocking (critical/high)

Домен этого репо — сама гейт-система; «деньги» здесь = целостность гейтов у всех
пользователей плагина:

- **Fail-open дыра.** Изменение, при котором гейт молча ПРОПУСКАЕТ то, что обязан
  блокировать: непроверенный деплой, код-коммит без лесенки, обход через конфиг/шим/env.
- **Ослабление через конфиг.** Всё, что позволяет `.codex-gate.yaml` (или его отсутствию/
  битости/незастейдженной правке) снять жёсткие код-пути, деплой-проверки или строгие
  дефолты. Строгий режим при нет/битом конфиге — инвариант.
- **Потеря fail-closed семантики.** Ошибка/outage/дрейф схемы вердикта, трактуемые как
  «чисто»; шим, исполняющий не-тот код или тихо пропускающий при отсутствии плагина.
- **Слом opt-in изоляции.** Плагин начинает вмешиваться в НЕ-онбордженные проекты
  (хуки не exit 0 вне git-репо / без конфига в worktree-OR-HEAD).
- **Молчаливый обход аудита.** Любой skip/обход без записи в audit-лог.
- **Тест, трогающий боевые файлы.** Тест, пишущий в реальные ledger'ы/маркеры/audit
  репо-носителя (изоляция-инварианты conftest).
- **Новый code path без теста, реально входящего в ветку.**

## Medium/low (совещательно)

Нет теста на новую ветку/except; сообщение об ошибке без контекста; пере-блок безобидного
(fail-safe трение); стиль/нейминг.

При сомнении в severity — меньшую, но опиши риск.

## Scope / известные остатки (НЕ флагать как blocking — осознанные решения)

Реестр спеки `docs/2026-07-22-gates-plugin-port-design.md` (Codex-approved, 4 раунда):

- **AGENTS.md вне жёстких код-путей** — наследованный by-design остаток (циклическая
  зависимость дизайн-гейта); ослабление видно в деплой-диффе; pin — Фаза 2.
- **G1 — session-scoped best-effort, cwd-rooted**: произвольный shell (`cp`/`mv`/
  `python -c`), вложенные репо, checkout до-онбординговой ревизии — вне охвата; дизайн-гейт
  fail-open by design (BS3-асимметрия), денежная защита — fail-closed деплой-цепочка.
- **Version-skew при обновлении плагина посреди коммита** — niche; ledger-tree
  валидируется на деплое.
- **Ladder self-attested**; rebase стирает ledger-покрытие (путь — audited LADDER_SKIP);
  кросс-машинные ledger/baseline локальны.

## Реестр остатков Codex-ревью (scope-решения проекта, НЕ пере-флагать)

- **empirical ML-E5 (2026-07-22, residual-failsafe → Фаза 2):** env-ослабление неизменной
  тест-команды (`PATH=/tmp/shims make deploy` подменяет `pytest` шимом; `PYTEST_ADDOPTS` и т.п.).
  НЕ чиним: тест-раннеру environment нужен (санитизация = fail-safe пере-блок), а оператор,
  задающий env на деплой, эквивалентен задающему `EMPIRICAL_SKIP` — защита от злонамеренного
  оператора вне охвата Фазы 1. Герметичный прогон в изолированном образе из отревьюенного
  коммита — Фаза 2 (тот же immutable-artifact-биндинг, что закрывает reviewed≡deployed). Порог
  эмпирического гейта: «поймать сломанные тесты + сделать конфиг-обходы аудируемыми», НЕ
  «остановить оператора с доступом к деплой-машине». Спека: docs/2026-07-22-empirical-gate-design.md ML-E5.
