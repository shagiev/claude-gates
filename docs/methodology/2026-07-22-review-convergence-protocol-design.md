# Протокол сходимости Claude↔Codex (finding-ledger + адъюдикация) — Фаза 1.6

**Дата:** 2026-07-22. **Статус:** design. **Мотиватор:** деплой боевого фикса — 8 раундов
гейта, 7 реальных багов, но выход только через человеческий `CODEX_REVIEW_SKIP` (гейт без
памяти между раундами: каждый прогон — свежий взгляд на растущий дифф, стена high'ов).
**Цель:** стоп-политика v2 (уже одобрена, спека Фазы 1 §«Стоп-политика») становится
МАШИННЫМ правилом внутри `check-reviewed`: Claude и Codex «договариваются», деплой проходит
сам, когда сходимость достигнута; несогласие эскалируется человеку одним вопросом, а не
бесконечным циклом. `CODEX_REVIEW_SKIP` остаётся только аварийным.

## Архитектура

### 1. Finding-ledger (`logs/review_findings/current.json`)
Один файл на деплой-серию (сбрасывается/архивируется при успешном деплое — baseline сдвинулся):
```json
{"baseline": "<sha>", "rounds": 3, "findings": {
  "F1": {"fingerprint": "app/db.py|bridge interval masks", "severity": "high",
          "title": "...", "status": "fixed|open|residual|refuted|duplicate",
          "reason": "...", "dup_of": null, "disputes": 0, "round": 2}}}
```
`rounds` инкрементится на каждый реальный прогон companion. Статусы меняет ТОЛЬКО
адъюдикация Claude (CLI) — кроме `duplicate` (авто по `[DUP:]`-префиксу).

### 2. Адъюдикация (CLI, шаг Claude между раундами)
`codex_review_gate.py findings` — показать текущие open.
`codex_review_gate.py adjudicate <Fid> fixed|residual|refuted "<причина>"` — классификация
по стоп-политике: `fixed` (закоммичен фикс), `residual` (fail-safe/niche/архитектура →
причина обязательна, дублируется в AGENTS-реестр вручную), `refuted` (опровергнуто
эмпирикой → доказательство в причине). Причина непустая — обязательна (аудит).

### 3. Codex видит адъюдикации (переговорная часть)
`_REVIEW_FOCUS` при непустом ledger дополняется блоком: «Previously adjudicated findings
(id, status, reason): ... If one of your findings restates item Fx, prefix its title with
[DUP:Fx]. If you dispute an adjudication with NEW evidence, prefix with [DISPUTE:Fx] and
state the evidence in body.» Схема плагина (`additionalProperties:false`) не позволяет
новое поле — сигнал в title-префиксе, парсится регекспом.

### 4. Обработка раунда (в `check_reviewed_cli`, codex-часть)
После парса вердикта: для каждой blocking-находки — `[DUP:Fx]`: если Fx open/fixed →
status=duplicate (привязка); если Fx residual/refuted → это ПЕРЕ-ПОДЪЁМ адъюдицированного
(Codex-фикс R1: DUP ≠ согласие) → `disputes[Fx]+=1`, Fx в open. Инструкция Codex'у явная:
«согласен с адъюдикацией → НЕ репорти находку вовсе; [DUP:] — только для open/fixed».
`[DISPUTE:Fx]` → `disputes[Fx]+=1`, Fx в open (несогласие = пере-открытие); иначе новый
`F<n>` со status=open. Затем машинное правило:
- **ALLOW** (exit 0, громкая сводка «сошлись: N fixed, M residual, K refuted»): нет open
  выше пола (blocking) — все находки последнего раунда адъюдицированы/дубли.
- **BLOCK** (exit 2, список open + инструкция адъюдицировать): есть open.
- **ESCALATE** (exit 2, спец-сообщение «нужен человек»): dispute на critical/high-находку
  с ПЕРВОГО раза (Codex-фикс R1: он — контролёр липовых refuted/residual; порог 1), на
  medium/low — с 2-го; ИЛИ `rounds > hard_cap` (8) при наличии open. Эскалация — штатный
  выход тупика, не скип. **Выход из ESCALATE (Codex-фикс R1):** (а) пользователь решает —
  `adjudicate <Fid> resolved-by-user "<решение>"` (записывается как решение человека,
  снимает open); (б) fix + новый раунд; (в) аварийный `CODEX_REVIEW_SKIP` (аудит). После
  (а) обычное правило: нет open → ALLOW.

### 5. Защита от rubber-stamping (Claude всё пометил residual)
(а) причина обязательна и аудируется; (б) residual-адъюдикации Codex видит в следующем
раунде и вправе `[DISPUTE:]` — двойной спор → человек; (в) residual требует ЯВНОГО направления:
CLI-статус — `residual-failsafe` (адъюдикатор утверждает: пере-блок/трение, НЕ пропуск
денег), просто `residual` не существует (Codex-фикс R1); для severity=critical
residual-failsafe запрещён вовсе (только fixed/refuted/эскалация); (г) любой липовый
refuted/residual контролирует Codex следующего раунда — его dispute на critical/high
эскалирует с первого раза (см. ESCALATE).

### 6. Сброс серии
Успешный деплой: `current.json` → `logs/review_findings/archive/<ts>-<baseline>.json`,
новый файл при следующей серии. Новый КОММИТ в серии сохраняет ledger (фиксы и есть
причина новых коммитов); `fixed`-статусы остаются (Codex их видит и не пере-флагает,
а если пере-флагает — это dispute с новым доказательством).

## BSAC
- **BS-C1:** деплой невозможен при непустом open выше пола. (тест)
- **BS-C2:** после адъюдикации всех open (residual/refuted/fixed с причинами) деплой
  проходит БЕЗ скипа. (тест)
- **BS-C3:** двойной dispute / hard-cap+open → эскалация (блок со спец-текстом), не
  тихий проход и не вечный цикл. (тест)
- **ML-C1:** rubber-stamp residual критической money-находки → запрещён CLI (только
  fixed/refuted/эскалация). (тест)
- **ML-C2:** `[DUP:]`-метка на самом деле новая находка → Claude при адъюдикации видит
  привязку и вправе пере-открыть (`adjudicate Fx open "не дубль"`). (CLI поддерживает)
- **ML-C3:** битый ledger-файл → fail-closed (блок, как отсутствие ревью). (тест)

## Тестирование
Юнит (парс префиксов, merge раунда в ledger, машинное правило allow/block/escalate,
critical-запрет residual, битый ledger); интеграционный: полный цикл на стабах companion
(раунд1 block → adjudicate → раунд2 c [DUP:] → allow).

## Остатки (осознанно)
- Codex может игнорировать префикс-конвенцию (не пометить дубль) → находка ляжет новым
  open → Claude адъюдицирует вручную (duplicate через `adjudicate Fx duplicate F3`) —
  трение, не дыра.

  автоматического семантик-матчинга нет (хрупко).
- Стоп-политика применяется к самому этому дизайну.

## Amendment A1 — carry-over (реш. юзера 22.07, после 13 живых раундов)

Паттерн «беговой дорожки»: каждый фикс растит дифф → новый раунд находит новый нюанс →
hard-cap → эскалация → фикс → … (худший случай — самореференция: ревью кода самого гейта).
**Правило:** после hard-cap НОВЫЕ неоспоренные не-critical находки текущего раунда →
`carried`: текущий деплой НЕ блокируют (громкая сводка + аудит), но стартуют СЛЕДУЮЩУЮ
серию открытыми — «бэклог с зубами»: следующий деплой на них заблокируется. Critical/
unknown-severity и оспоренные не переносятся. Анти-гниение: находка, пережившая 2 серии
(carry_count≥2), дальше не переносится — эскалация человеку. Судьбы: fixed (след. серия) |
residual-failsafe → AGENTS-реестр | архитектурное → спека Фазы 2.
