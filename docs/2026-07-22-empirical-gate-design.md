# Эмпирический гейт деплоя (тесты как условие allow) — дизайн

**Дата:** 2026-07-22. **Статус:** design (на Codex-ревью). **Тикет:** #1. **Tier:** 2
(actuator-safety — guard над деплой-актуатором; провал в fail-open = непроверенный деплой).

## Мотивация

SOTA-ресёрч SDD (2026-07-22): единогласный консенсус 80+ ревью-агентов подтвердил
несуществующую уязвимость — убил её один эмпирический тест (arxiv 2604.19049). Вывод:
суждение ревьюера (даже независимого Codex) не заменяет прогон тестов. adversarial-review
(ng) ставит механические проверки первыми и бесплатно ДО дорогого LLM-ревью. У нас
деплой-гейт судит дифф Codex'ом, но не требует, чтобы тесты проекта были зелёными.

## Что делаем

В `check_reviewed_cli` добавляется ЭМПИРИЧЕСКИЙ гейт: прогон сконфигурированной
тест-команды проекта. Место в порядке (cheap→dear, SOTA): clean-tree → baseline →
ladder → **empirical** → Codex. Тесты падают → блок ДО вызова Codex (токены не тратятся).

Конфиг в `.codex-gate.yaml` (корень целевого репо):
```yaml
empirical:
  test_command: "python3 -m pytest -q"   # opt-in; нет → скип с громкой пометкой
  timeout_s: 600                          # опц., дефолт 600; невалидное → дефолт
```

## Scenario-first: пространство входов (ТЕКСТ до кода)

Конфиг читается ПРИВЯЗАННО К SHA — `git show <ref>:.codex-gate.yaml` (не worktree): для
HEAD — от захваченного `head_before` (см. R1-F2), для baseline — от `baseline`. Валидная
команда ref'а = непустой `test_command` (str) в валидном YAML; иначе `None`.

Измерения:
- **D1 команда на HEAD** (`head_cmd`): {None(нет секции / битый YAML / нет PyYAML / test_command не-str/пустой), валидная}
- **D2 команда на baseline** (`base_cmd`): {None, валидная} — «был ли гейт включён РАНЬШЕ»
- **D3 результат команды**: {exit 0, exit≠0, не запустилась (FileNotFoundError/OSError), таймаут}
- **D4 `EMPIRICAL_SKIP`**: {не задан, "1"}
- **D5 `timeout_s`**: {отсутствует→дефолт, валидный, невалидный(не-int/<1)→дефолт}
- **D6 HEAD/дерево во время команды**: {неизменны, HEAD сменился, дерево загрязнилось}

**Ключевое правило (R1-F1, анти-fail-open):** «нет валидной команды» трактуется по тому,
**был ли гейт включён в baseline**:
- `head_cmd=None` И `base_cmd=None` → гейт никогда не был opt-in → **скип** (легитимно).
- `head_cmd=None` И `base_cmd≠None` → гейт БЫЛ включён, теперь удалён/сломан/испорчен →
  **БЛОК** (fail-closed), кроме аудируемого `EMPIRICAL_SKIP`. Отключение гейта = скип гейта:
  оба требуют одного осознанного аудируемого действия. Порча YAML/удаление секции/подмена на
  `true` больше не проходят тихо (закрывает связку `CODEX_REVIEW_SKIP`+порча).
- `head_cmd≠None` → команда бежит (D3).

Сценарии (= тест-матрица):

| # | Условие | Ожидание |
|---|---|---|
| S1 | head=None, base=None | скип + ГРОМКАЯ пометка stderr, деплой продолжается (никогда не opt-in) |
| S2 | head≠None, D3=exit0 | гейт пройден, переход к Codex |
| S3 | head≠None, D3=exit≠0 | **БЛОК (2)** + хвост вывода команды — денежный fail-closed |
| S4 | head≠None, D3=не запустилась | **БЛОК (2)** — «не запустилось» ≠ «прошло» (silent-reject аналог) |
| S5 | head≠None, D3=таймаут | **БЛОК (2)** после `timeout_s` |
| S6 | D4="1" | скип + аудит-строка, деплой продолжается (перекрывает любой блок ниже) |
| S7 | head=None (битый YAML/нет секции), base≠None | **БЛОК (2)** — гейт был включён, теперь сломан/снят (R1-F1) |
| S8 | head=None (битый YAML), base=None | скип — гейт никогда не был включён |
| S9 | D5=невалидный | дефолт 600, команда всё равно бежит |
| S10 | head≠None, D3=exit≠0 + ordering | Codex НЕ вызывается (стаб, падающий при вызове, не тронут) |
| S11 | `CODEX_REVIEW_SKIP=1` + head≠None, D3=exit≠0 | **БЛОК** — эмпирика независима от Codex-скипа |
| S12 | head≠None, D3=exit0, D6=HEAD сменился | **БЛОК (2)** — тест не для задеплоенного SHA (R1-F2) |
| S13 | head≠None, D3=exit0, D6=дерево загрязнилось | **БЛОК (2)** — дерево ≠ проверенному (R1-F2) |

Проверка полноты по сетке: каждое значение D1–D6 встречается ≥1 раз; ключевые комбинации —
was-on×now-off (S7 vs S8), ordering (S10), независимость от Codex-скипа (S11),
escape перекрывает всё (S6), race HEAD/дерева (S12/S13).

Осознанно ВНЕ scope: флейки/ретраи (дело стабильной команды пользователя), coverage-порог
(внутри тест-команды), запуск в контейнере/изоляции (бежим в cwd репо, как деплой), инкрементальный
выбор тестов (гоняем всю команду).

## EARS — приёмочные критерии (→ тесты)

- **EARS-1:** WHEN `head_cmd` задан И exit 0 — гейт SHALL перейти к Codex-шагу. (S2)
- **EARS-2:** IF команда exit≠0 — THEN гейт SHALL заблокировать деплой (2) и напечатать хвост вывода. (S3)
- **EARS-3:** IF команда не запустилась (не найдена/OSError) ИЛИ таймаут — THEN гейт SHALL
  заблокировать деплой (fail-closed). (S4, S5)
- **EARS-4:** WHERE `head_cmd=None` И `base_cmd=None` (гейт никогда не был opt-in) — гейт SHALL
  пропустить проверку с громкой однострочной пометкой и продолжить. (S1, S8)
- **EARS-5:** WHILE `EMPIRICAL_SKIP=1` — гейт SHALL пропустить проверку, записать аудит-строку,
  продолжить (перекрывает любой блок). (S6)
- **EARS-6:** Эмпирическая проверка SHALL идти ПОСЛЕ ladder-range и ДО Codex-ревью. (S10)
- **EARS-7:** `CODEX_REVIEW_SKIP=1` SHALL NOT пропускать эмпирическую проверку. (S11)
- **EARS-8:** IF `head_cmd=None` НО `base_cmd≠None` (гейт был включён в baseline, теперь
  удалён/сломан/испорчен) — THEN гейт SHALL заблокировать деплой (fail-closed), кроме
  `EMPIRICAL_SKIP`. (S7) — отключение гейта требует того же аудируемого действия, что и скип.
- **EARS-9:** Конфиг (HEAD и baseline) SHALL читаться привязанно к SHA (`git show <ref>:…`),
  не из worktree.
- **EARS-10:** IF за время прогона команды HEAD сменился ИЛИ дерево загрязнилось — THEN гейт
  SHALL заблокировать деплой (тест был не для задеплоенного состояния). (S12, S13)

## BSAC — бизнес-сценарии и денежные краевые случаи

Бизнес-сценарии:
- **BS-E1:** деплой с падающими тестами проекта не проходит (кроме аудируемого `EMPIRICAL_SKIP`). (S3)
- **BS-E2:** opt-in — репо без `empirical.test_command` не затронуты. (S1)
- **BS-E3:** обход только осознанный и со следом в аудите. (S6)

Денежные/fail-open краевые случаи:
- **ML-E1** команда не запустилась → трактуется как «прошло» → непроверенный деплой. →
  S4/S5 fail-closed (актуатор-урок: «verify it LANDS» — подтвердить, что команда реально
  отработала, а не молча ничего).
- **ML-E2 [R1-F1]** удаление секции / подмена `test_command` на `true` / порча YAML, чтобы
  снять гейт. → **правило base-vs-head (EARS-8)**: если `base_cmd≠None`, а `head_cmd=None` —
  блок, кроме `EMPIRICAL_SKIP`. Отключение гейта = скип гейта, оба через один аудируемый
  канал. Аргумент «увидит Codex» НЕ полагается (связка `CODEX_REVIEW_SKIP`+порча его обходит);
  защита самодостаточна в эмпирическом гейте.
- **ML-E3 [R1-F2]** смена HEAD за время прогона (тест ≤600с): чистый HEAD A → тест зелёный →
  параллельный commit/checkout ставит чистый B → Codex смотрит B, финальная проверка видит
  стабильный B, allow; B эмпирику не проходил. → `head_before` захватывается после clean-tree;
  после команды перепроверка `HEAD==head_before` И `working_tree_clean()`, иначе блок
  (S12/S13). ПОПРАВКА к прежней версии: утверждение про deploy-lock во время эмпирики было
  НЕВЕРНО — Makefile берёт лок в рецепте ПОСЛЕ prerequisite `check-reviewed`, т.е. сам гейт
  бежит без лока; связывание с SHA — правильная защита, не лок.
- **ML-E4** таймаут-вечная команда. → `timeout_s` (дефолт 600) + fail-closed по таймауту (S5).

## Actuator-safety чек-лист (Tier 2)

- **Независимый монитор исхода:** сам гейт наблюдаемый — exit-код + stderr-хвост + (при скипе)
  аудит; провал/скип не тихий.
- **Kill-switch/escape:** `EMPIRICAL_SKIP=1` (аудируемый), в ряду с LADDER_SKIP/CODEX_REVIEW_SKIP;
  общий деплой kill-switch — freeze проекта (вне этого гейта).
- **Ограниченные полномочия:** авторитет `test_command` ограничен тем, что он закоммичен и
  виден ревью диффа (жёсткий код-путь); на чистом дереве worktree==HEAD.
- **Recovery:** гейт НЕ мутирует внешнее состояние — откат не нужен; блок деплоя не трогает
  уже работающий актуатор.

Определение готово: реальный маршрут воспроизведён — падающая тест-команда РЕАЛЬНО блокирует
`check_reviewed_cli` (не только юнит хелпера), и Codex при этом не вызван.

## Отвергнутые альтернативы

- **Required (блок, если тест-команда не задана):** сломал бы репо без тестов; opt-in +
  громкая пометка честнее. Строгий-по-умолчанию тут = трение без пользы (fail-safe, не money).
- **Встроить в скилл design-review, а не в деплой-гейт:** дизайн-гейт fail-open by design
  (мышление не стопорится), а эмпирика — про НЕ-деплой непроверенного; её место — деплой-гейт
  (fail-closed).
- **Эмпирика ПОСЛЕ Codex:** тратит Codex-токены на код, который и так падает; SOTA прямо
  советует механику первой.
- **Свой парсер типов ошибок команды (различать flaky/real):** хрупко; «не exit 0 = блок»
  проще и безопаснее, ретраи — дело стабильной команды пользователя.

## Тестирование

Юнит (изоляция как в conftest; тест-команда — стаб-скрипты `bash -c 'exit N'` / `sleep`):
S1–S10 из матрицы. Отдельно: ordering (S9) — Codex-стаб, падающий при вызове, не должен быть
тронут при провале эмпирики; независимость от `CODEX_REVIEW_SKIP` (S10). Валидация `timeout_s`
(S8). Битый `empirical` (S7). Все — через `check_reviewed_cli` (реальный маршрут), не только
хелпер.
