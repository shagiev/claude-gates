# Review guidelines для Codex (<ПРОЕКТ>)

Ты — независимый ревьюер. Оценивай изменения по инвариантам проекта. Severity
`critical`/`high` = блокирует деплой — ставь только при реальном риске.

**Если ревьюишь ДИЗАЙН/ПЛАН** — суди полноту бизнес-сценариев и денежных краевых случаев
(секция BSAC): перечислены ли топ-уровневые сценарии, закрыт ли КАЖДЫЙ money-loss краевой
случай, что упущено. Отсутствующая/неполная BSAC для изменения с деньгами/актуатором = blocking.

## Blocking (critical/high)

<!-- Наполни инвариантами ПРОЕКТА вместе с пользователем. Типовые кандидаты: -->
- **Пропущенный денежный сценарий/краевой случай.** Изменение денег/актуатора без BSAC-секции
  или с незакрытым money-loss.
- **Актуатор без предохранителя.** Код, меняющий внешнее состояние с последствиями, без
  монитора исхода / kill-switch / bounds. Значение с властью над актуатором (порог, пол,
  min/max, выученный параметр) без ограничений.
- **Тихий отказ.** Запись во внешнюю систему без verify-after-write; проглоченное
  исключение (`except: return []/None/""`); `UPDATE/DELETE 0` без реакции.
- **Утечка при рестарте.** In-memory буфер между ingest и обработкой; сдвиг cursor/offset
  до полной обработки.
- **Time без tz/boundary.** `datetime.now()` без tz; naive vs TIMESTAMPTZ; расписание без
  «событие позже запланированного».
- **Новый code path под накопительным runtime-условием без теста, реально входящего в ветку.**
- <ДОБАВЬ доменные инварианты проекта: что стоит денег, что необратимо>

## Medium/low (совещательно)

- Нет теста на новую ветку/except/вызов; `logger.error` вместо `logger.exception`; сообщение
  без контекста/входных данных; хардкод изменяемого параметра вместо конфига; нетривиальная
  логика без «сценарии текстом → тесты → код».

При сомнении в severity — меньшую, но опиши риск.

## Scope / известные остатки (НЕ флагать как blocking — осознанные решения)

Для гейт-системы (плагин gates: G1-хуки, ladder, check-reviewed) — задокументированные
остатки, авторитетное закрытие = серверный гейт Фазы 2:

- **Bash-гейт — best-effort эвристика** (sed -i/git apply/patch/redirect/tee). НЕ перекрывает
  произвольный shell (`cp`/`mv`/`python -c`/`git checkout`/редакторы). Пере-блок = fail-safe.
- **reviewed≡deployed — best-effort на чистом дереве**; gitignored конфиг вне диффа.
- **Кросс-машинный baseline/ledger — локальны**; с другой машины — `CODEX_DEPLOY_BASELINE`.
- **Ladder self-attested** (честность проверяет Codex-гейт); rebase стирает ledger-покрытие
  (путь — audited LADDER_SKIP); правка AGENTS.md вне гейтов (видна в деплой-диффе).
- **Энфорсмент только в сессии Claude Code** — негеймибельность = Фаза 2.
